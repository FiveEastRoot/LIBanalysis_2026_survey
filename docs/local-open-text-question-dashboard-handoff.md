# 노원구 로컬 서술형 문항 대시보드 작업 인계

작성일: 2026-07-14

## 목적

서울도서관 제출 기준 문항과 별개로, 노원구 조사폼 마지막에 로컬 확장 서술형 문항 1개를 추가한다. 이 문항은 서울도서관 공식 문항번호 체계에는 포함하지 않고, LIBanalysis/노원구 내부 검토용 응답으로 저장한다.

## 추가 문항

| 항목 | 값 |
| --- | --- |
| 문항 성격 | 노원구 로컬 확장 서술형 |
| 문항 위치 | 조사폼 마지막, `도서관 제안` 별도 섹션 |
| 조사폼 섹션 | `local_feedback` / 도서관 제안 |
| 문항 유형 | `long_text` |
| 필수 여부 | 선택 권장 |
| 문항 문구 | 그 밖에 도서관에 제안하고자 하는 내용을 자유롭게 적어주세요. |
| 서울도서관 문항번호 | `LOCAL` 또는 빈값 |
| LIBanalysis 문항번호 | `NW-OE-1` 권장 |
| 분석 1차 반영 | 제외 |
| 원자료 저장 | 포함 |
| 개인정보 주의 | 서술형 내 개인정보 입력 가능성 안내 필요 |
| 입력 제한 | 최대 300자 |

## 권장 저장/Export 규칙

| 구분 | 권장 처리 |
| --- | --- |
| DB 저장 | `analysis_payload`에 `NW-OE-1` 또는 별도 로컬 확장 key로 저장 |
| CSV 위치 | 전체 원자료 CSV의 마지막 컬럼 |
| 서울시 제출용 export | 제외하거나 `LOCAL` 문항으로 명확히 분리 |
| LIBanalysis 분석용 export | 포함 가능하되 기본 분석 계산에는 사용하지 않음 |
| 파일 업로드 검증 | 허용 문항 목록에 `NW-OE-1` 추가 |
| DB 조회 검증 | Supabase 원천 조회와 CSV 업로드 양쪽에서 동일하게 허용 |
| 결측 처리 | 빈 문자열 허용 |
| 최대 글자 수 | 300자 |

## 대시보드 세션에서 해야 할 작업

1. 문항 매핑 정의에 `NW-OE-1` 추가
   - 서울도서관 문항번호: `LOCAL` 또는 빈값
   - LIBanalysis 문항번호: `NW-OE-1`
   - 문항명: `그 밖에 도서관 제안사항`
   - 원문 문항: `그 밖에 도서관에 제안하고자 하는 내용을 자유롭게 적어주세요.`

2. 업로드 검증 엔진 수정
   - CSV 업로드 시 `NW-OE-1`을 허용 컬럼으로 처리
   - 알 수 없는 컬럼 경고에서 제외
   - 값이 비어 있어도 오류로 보지 않음
   - 300자를 초과한 값은 검증 경고 또는 잘림 정책을 명확히 적용

3. DB 원천 조회 처리 수정
   - `survey_analysis_export.analysis_payload` 안에 `NW-OE-1`이 있을 때 무시하지 않고 원자료 row에 포함
   - 기존 DB 조회 방식과 CSV 업로드 방식이 동일한 normalized row를 만들도록 조정

4. Export 처리 수정
   - 전체 원자료 CSV에 `NW-OE-1` 마지막 컬럼 추가
   - 서울도서관 문항번호와 LIBanalysis 문항번호를 함께 쓰는 long format export가 있다면 다음처럼 처리
     - `seoul_question_code`: `LOCAL`
     - `libanalysis_question_code`: `NW-OE-1`
     - `response_value`: 응답 원문

5. 대시보드 UI 처리
   - 기본 분석/심화 분석 지표에는 사용하지 않음
   - 필요 시 프로젝트 상세 또는 원자료 확인 영역에서 텍스트 응답 목록으로만 표시
   - 개인정보가 포함될 수 있으므로 일반 공개형 요약/AI 입력에는 넣지 않는 것을 기본값으로 둠

6. 보안/개인정보 처리
   - 서술형 응답에 이름, 전화번호, 주소 등 개인정보가 입력될 수 있음
   - AI 요약 또는 보고서 생성에 넣기 전 개인정보 필터링/마스킹 정책 필요
   - 원자료 다운로드 권한 범위는 기존 role 기반 권한을 따름
   - 폼 입력 단계에서는 300자 제한을 적용하며, 대시보드 업로드/DB 조회 단계에서도 동일 정책을 검증 기준으로 둠

## 조사폼 세션에서 해야 할 작업

1. `questionSchema.ts` 마지막에 `NW-OE-1` 서술형 문항 추가 완료
2. `독서` 섹션이 아니라 `도서관 제안` 별도 섹션으로 분리
3. 선택 문항이므로 전체 진행률 계산에서는 제외
4. 제출 payload와 원자료 export에는 마지막 항목으로 포함
5. 문항 매핑 문서와 Notion 문서 업데이트

## 대시보드 세션 전달 요약

조사폼 마지막에 노원구 로컬 확장 서술형 문항 `NW-OE-1`을 추가했다. 문항 문구는 `그 밖에 도서관에 제안하고자 하는 내용을 자유롭게 적어주세요.`이다. 이 문항은 `독서` 섹션이 아니라 `도서관 제안` 별도 섹션으로 관리한다. 서울도서관 제출 기준 문항이 아니므로 서울도서관 문항번호는 `LOCAL` 또는 빈값으로 처리하고, LIBanalysis 기준 문항번호 `NW-OE-1`로 관리한다. 대시보드는 CSV 업로드와 DB 원천 조회 양쪽에서 이 필드를 허용해야 하며, 기본 분석 계산에는 포함하지 않고 원자료/텍스트 검토용으로만 다루는 것을 권장한다.
