## 작업 목표

조사폼 마지막에 노원구 로컬 확장 서술형 문항이 추가되어 있으므로, 대시보드의 CSV 업로드, DB 원천 조회, 원자료 저장/export 흐름에서 이 문항을 정상적으로 허용하고 보존하게 수정해줘.

## 추가된 문항

| 항목 | 값 |
| --- | --- |
| 문항코드 | `NW-OE-1` |
| 문항 유형 | `long_text` |
| 문항 위치 | 조사폼 마지막 |
| 조사폼 섹션 | `local_feedback` / 도서관 제안 |
| 필수 여부 | 선택 |
| 문항 문구 | 그 밖에 도서관에 제안하고자 하는 내용을 자유롭게 적어주세요. |
| 서울도서관 문항번호 | `LOCAL` 또는 빈값 |
| LIBanalysis 문항번호 | `NW-OE-1` |
| 기본/심화 분석 반영 | 제외 |
| 원자료 저장/export | 포함 |
| 입력 제한 | 최대 300자 |

## 구현 요청

1. 문항 매핑/검증 기준에 `NW-OE-1`을 추가해줘.
   - 이 문항은 `독서` 섹션이 아니라 `도서관 제안` 별도 섹션으로 분류한다.
2. CSV 업로드 방식과 Supabase DB 조회 방식이 동일한 normalized row를 만들도록 맞춰줘.
3. 전체 원자료 CSV에는 `NW-OE-1`을 마지막 컬럼으로 포함해줘.
4. 서울도서관 제출용 export와 분석용 export를 분리해서, 서울도서관 공식 문항 체계가 깨지지 않게 처리해줘.
5. long format export가 있다면 `seoul_question_code=LOCAL`, `libanalysis_question_code=NW-OE-1`, `response_value=응답 원문` 형태로 처리하는 방식을 우선 검토해줘.
6. 이 문항은 기본 분석/심화 분석 지표에는 넣지 말고, 원자료 확인 또는 별도 텍스트 검토 영역에서만 다뤄줘.
7. 서술형 응답에 개인정보가 들어갈 수 있으므로 AI 요약, 보고서 생성, 공개형 화면 투입 전에 마스킹/제외 정책을 적용해줘.
8. 폼 입력 제한은 300자이며, CSV 업로드와 DB 조회 검증에서도 300자 초과 값에 대한 경고 또는 처리 정책을 적용해줘.

## 주의사항

- 서울도서관 제출 기준 export 구조는 유지해야 한다.
- `NW-OE-1`은 노원구 로컬 확장 문항이며, 서울도서관 원문 문항번호가 아니다.
- 값이 비어 있어도 오류로 처리하지 않는다.
- CSV 업로드와 DB 조회 양쪽에서 동일하게 허용해야 한다.
- 대시보드 권한과 원자료 다운로드 권한은 기존 role 기반 정책을 따른다.
